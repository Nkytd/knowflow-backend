# 前端管理台布局审查记录

## 本轮审查范围

本轮审查覆盖 `src/main/resources/static/console` 下全部管理台和用户侧页面：

- 运营看板
- 智能问答入口
- 知识运营 / 文档管理 / 解析任务
- 问答记录 / 工单管理 / 知识草稿
- 运维健康 / 死信治理 / 审计日志
- 用户中心

## 发现的问题

### 1. 全局导航与模块子导航混在一起

原先公共导航会把当前模块下的子页面直接追加到全局导航后面。

例如在知识运营模块里，导航会连续出现：

```text
运营看板 / 智能问答入口 / 知识运营 / 工单协同 / 系统治理 / 文档管理 / 解析任务 / 全部页面 / 用户信息
```

问题是用户难以区分哪些是一级模块，哪些是当前模块下的二级页面。

本轮调整后：

```text
运营看板 / 智能问答入口 / 知识运营 / 工单协同 / 系统治理 / 当前 文档管理 / 全部页面 / 用户信息
```

当前页面胶囊点击后会打开“模块页面”菜单，二级页面统一放进下拉层，不再污染主导航。

### 2. 双栏页面对齐不稳定

知识库、文档、解析任务、问答记录、工单、草稿、死信、审计等页面都使用左列表 + 右详情结构。

原先默认布局使用固定左列宽度，且两侧面板高度不拉齐，容易出现：

- 左右卡片顶部或底部视觉不齐
- 表格内容变宽后挤压详情区
- 筛选项高度不同导致查询按钮位置不一致

本轮统一了普通 `workspace-grid` 的列宽、拉伸、最小宽度和工具栏底部对齐规则。

同时把文档管理、解析任务、知识草稿页的筛选区瘦身：筛选区只保留真正参与查询的条件，刷新时间移动到操作区，能力说明不再挤占筛选栅格。

### 3. 大说明框占用首屏空间

工单管理、解析任务、知识草稿、死信治理、审计日志等页面仍保留正文顶部的 `hero-panel` 说明框，导致进入页面后主列表和详情区下移。

本轮将这些说明统一迁移到页面标题悬浮说明中：鼠标移到标题即可查看页面定位、演示亮点和业务闭环说明，正文首屏则优先展示真实业务内容。

### 4. 页面缓存容易导致“改了但看不到”

多数页面引用 `workbench.css` 时没有版本号，浏览器可能继续使用旧样式。

本轮统一给公共 CSS / JS 引用加了 `layout-audit-20260430b` 版本号，减少调试时的缓存干扰。

## 当前布局规范

- 一级导航只展示五个业务模块：运营看板、智能问答入口、知识运营、工单协同、系统治理。
- 当前模块的二级页面统一收纳到“当前页面”胶囊下拉菜单。
- “全部页面”只负责跨模块完整入口，不再承载当前模块说明。
- 用户菜单固定在导航最右侧。
- 普通左右分栏页面默认两栏拉齐，移动端自动折叠为单栏。
- 页面说明优先放到标题悬浮说明中，不再额外占据正文首屏空间。

## 后续建议

- 增加一个前端视觉回归脚本，自动截图关键页面并对比导航高度和首屏布局。
- 将管理台静态页逐步抽象为模板或组件，减少 HTML 页面之间的重复。
- 给列表 + 详情页面增加统一的“空状态、选中态、深链高亮态”组件规范。

## 2026-05-08 交互细节补充

本轮继续围绕“全页面设计不合理的小摩擦”做共性修复：

- 模块内页面下拉不再粗暴移除模块首页。当前页面仍不会重复出现，但在 `文档管理`、`解析任务` 等子页里可以直接回到 `知识库管理`，不用只依赖一级导航。
- `common.js` 新增全局来源返回条，自动识别 `returnUrl` / `from` 参数，并只允许同站内路径，避免开放跳转风险。
- 账户菜单里的“用户中心”继续携带 `from` 来源，用户从任意页面进入个人资料后，可以回到原页面。
- `common.js` 新增 `renderStateBlock()`，作为统一的空态、加载态、错误态生成器。
- `documents.js` 和 `qa-records.js` 已先接入统一状态块，覆盖列表加载、列表空态、详情加载、详情错误和未选中提示。
- 全部页面公共 CSS / JS 版本号提升到 `interaction-polish-20260508`，避免浏览器缓存旧导航和旧状态样式。

### 2026-05-08 状态块收口

已继续按模块把剩余页面的主列表、详情、加载、错误和未选中提示接入 `renderStateBlock()`：
