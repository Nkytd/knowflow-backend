# 求职项目包装指南

## 项目一句话定位

KnowFlow 是一个企业级智能知识服务与工单协同平台，打通了知识库问答、未命中转人工、工单解决后知识回流、运营分析看板的完整闭环。

## 简历写法

可以直接写成 3 条：

- 负责设计并实现多租户知识服务平台后端，基于 Spring Boot、MyBatis-Plus、JWT、Flyway 构建租户、用户、角色、知识库、文档、工单等核心业务模块。
- 搭建文档解析与向量索引异步流水线，接入 RabbitMQ、Redis、MinIO，支持文档上传、解析、chunk 化、向量化、失败重试、死信治理。
- 实现混合检索问答与知识回流闭环，支持问答未命中转工单、工单解决生成知识草稿、审核发布回流知识库，并提供运营看板和检索评估脚本。
- 补充企业级治理能力，实现工单 SLA 超时扫描、AI Token/成本统计、Agent 工具调用接口，支持查询工单进度、创建工单、查询知识库和生成知识草稿等受控业务动作。

## 面试重点

建议把讲述重点放在三件事上：

1. 这不是普通 CRUD，而是完整业务闭环  
2. 这不是单纯调大模型，而是做了检索、异步任务和治理能力  
3. 这不是只做功能，而是补了工程化、测试和可评估能力

## 技术亮点讲法

### 1. 业务价值

- 降低重复问答的人力成本
- 把“未命中问题”转成知识库建设输入
- 帮运营和知识管理员定位知识缺口

### 2. 后端能力

- 多租户隔离
- JWT 鉴权链路
- 文档解析任务状态机
- 工单与知识草稿的跨模块联动
- 审计日志与失败治理

### 3. AI 应用能力

- 文档切 chunk
- 向量化索引
- 词法 + 向量混合检索
- 低置信度转人工
- 知识回流形成 RAG 闭环

## 面试官常见追问

### 问：为什么这个项目有竞争力？

答法：

它同时覆盖了传统 Java 后端、企业业务建模、异步任务编排和 AI 应用落地，不是停留在简单聊天页，而是有完整业务闭环和工程化能力。

### 问：你做的最有技术含量的部分是什么？

答法：

我重点做了知识检索和异步处理链路。前者解决“能不能回答得准”，后者解决“系统能不能稳定跑起来”，两块加起来才像一个真实系统。

### 问：这个项目还可以继续怎么演进？

答法：

下一步会继续补检索评估数据集、接入专业向量数据库、补监控告警和限流熔断，让它从可演示继续逼近生产系统。
