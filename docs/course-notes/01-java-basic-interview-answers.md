## 1. `JDK`、`JRE`、`JVM` 的关系是什么？

### 简答版

JVM 是 Java 虚拟机，负责执行字节码；JRE 是运行环境，包含 JVM 和核心类库；JDK 是开发工具包，包含 JRE 以及 `javac` 等开发工具。所以关系可以理解为 `JDK > JRE > JVM`。

### 面试表达版

“JVM 是 Java 程序真正运行的地方，负责加载和执行字节码。JRE 是运行 Java 程序所需的环境，里面除了 JVM 还有核心类库。JDK 则是在 JRE 的基础上增加了编译器和调试工具，适合开发使用，所以如果我要写 Java 程序，我装的是 JDK，而不是只装 JRE。”

## 2. Java 为什么能跨平台？

### 简答版

因为 Java 源码会先编译成平台无关的字节码，不同操作系统上有各自的 JVM，JVM 再把同一份字节码转换成对应平台的机器指令。

### 面试表达版

“Java 的跨平台不是说源码直接到处跑，而是说源码会先编译成 `.class` 字节码。这个字节码本身和平台无关，真正和平台相关的是 JVM。Windows、Linux、Mac 上都有各自实现的 JVM，所以同一份字节码可以在不同平台的 JVM 上运行。”

## 3. Java 为什么既像编译型语言又像解释型语言？

### 简答版

因为 Java 源码会先经过 `javac` 编译成字节码，这是编译型的一面；字节码再交给 JVM 解释执行，这是解释型的一面；同时 JVM 还会用 JIT 对热点代码做即时编译优化。

## 4. `==` 和 `equals()` 的区别是什么？

### 简答版

- 对基本类型，`==` 比较的是值
- 对引用类型，`==` 比较的是地址
- `equals()` 通常比较的是对象内容

### 面试表达版

“`==` 对基本类型比较的是值本身，比如两个 `int` 是否相等；对引用类型比较的是两个引用是否指向同一个对象。`equals()` 则更常用于比较对象内容是否相等，比如两个字符串内容是否一致。实际开发中，比较字符串内容通常用 `equals()`，而不是 `==`。”

## 5. 为什么 `String` 是不可变的？

### 简答版

因为字符串对象创建后内容不能修改，这样更安全，也方便字符串常量池复用，还更适合作为 `HashMap` 的 key。

### 面试表达版

“`String` 设计成不可变，主要是出于安全性和性能考虑。不可变对象天然更安全，不容易被意外修改；同时 Java 可以复用字符串常量池，提高内存利用率；另外不可变也让它更适合作为哈希结构中的 key，因为内容不会变化，`hashCode` 更稳定。”

## 6. `StringBuilder` 和 `StringBuffer` 的区别是什么？

### 简答版

- `StringBuilder`：可变、非线程安全、性能更好
- `StringBuffer`：可变、线程安全、性能相对低一点

### 面试表达版

“如果是单线程环境下频繁拼接字符串，我一般优先用 `StringBuilder`，因为它效率更高；如果确实存在多线程并发修改同一个字符串缓冲区的场景，才考虑 `StringBuffer`，因为它是线程安全的。”

## 7. 为什么金额计算一般不用 `double`？

### 简答版

因为 `double` 是浮点数，某些十进制小数无法被精确表示，容易产生精度误差。金额、价格这类场景一般用 `BigDecimal`。

### 面试表达版

“`double` 底层是二进制浮点表示，像 `0.1`、`0.2` 这样的十进制小数在二进制里不一定能精确表示，所以做金额运算容易有误差。企业开发里金额、价格、精确统计这类场景通常会用 `BigDecimal` 来保证精度。”

## 8. Java 中为什么要有泛型？

### 简答版

泛型主要是为了类型安全，让问题尽量在编译期暴露，而不是拖到运行期。

### 面试表达版

“泛型的核心价值是类型安全。比如 `List<String>` 明确表示这个列表里只能放字符串，编译器会帮我做检查，这样能减少运行期的类型转换错误。同时泛型也能让代码可读性更好，别人一看就知道集合里存的是什么类型。”

## 9. `List`、`Set`、`Map` 的使用场景分别是什么？

### 简答版

- `List`：有序、可重复，适合保存一批有顺序的数据
- `Set`：不允许重复，适合去重
- `Map`：键值对结构，适合通过 key 快速查 value

### 面试表达版

“如果我要保存一批有顺序的数据，比如文档列表、检索结果列表，我一般用 `List`；如果重点是去重，比如标签集合、id 去重，我会用 `Set`；如果我要通过某个 key 快速查值，比如根据用户 id 查用户名，我会用 `Map`。”

## 10. 什么是重载，什么是重写？

### 简答版

- 重载：同一个类里，同名但参数不同
- 重写：子类对父类方法进行重新实现

### 面试表达版

“重载发生在同一个类中，方法名相同，但参数列表不同；重写发生在继承关系中，子类对父类已有方法提供自己的实现。重载更偏向同一能力的不同调用方式，重写更偏向多态和扩展。”

## 11. 抽象类和接口怎么区分？

### 简答版

- 抽象类更像模板
- 接口更像规范

### 面试表达版

“如果我有一类对象，它们之间有明显共性，并且我还想复用一部分公共实现，我会更倾向抽象类；如果我更关心的是某种能力规范，比如谁实现了这个接口就具备这种能力，那我会更倾向接口。简单说，抽象类更偏模板，接口更偏规范。”

## 12. Java 里到底有没有引用传递？

### 简答版

Java 只有值传递。对象参数传递的是引用值的副本。

### 面试表达版

“Java 只有值传递，不存在真正意义上的引用传递。对于对象参数来说，传进去的是引用值的副本。因为这个副本和外部引用指向同一个对象，所以在方法里改对象内容，外部能看到；但如果在方法里把参数重新指向一个新对象，外部是不会跟着变的。”

## 13. 运行时异常和受检异常有什么区别？

### 简答版

- 运行时异常通常表示程序逻辑问题或非法状态
- 受检异常通常表示某些外部操作可能失败，编译器要求显式处理

### 面试表达版

“运行时异常比如空指针、数组越界，更偏代码逻辑问题；受检异常比如 `IOException`，通常代表外部资源操作可能失败，编译器要求开发者显式处理。实际开发里，这两类异常的使用场景和处理方式不太一样。”

## 14. 为什么项目里要统一用 UTF-8？

### 简答版

因为不同系统和工具的默认编码可能不一样，如果不统一，中文内容容易乱码。

### 面试表达版

“统一 UTF-8 主要是为了避免编码不一致导致的乱码问题。比如文件是 UTF-8，但程序按 GBK 去读，就很容易出现中文异常。尤其是后端项目涉及文件读写、日志、接口传输时，统一编码是很重要的工程习惯。”
