# Docker + MinIO + 本地部署知识框架

## 1. 为什么你必须学 Docker

你已经亲身体验过：

- 本地环境依赖很多
- MySQL、Redis、RabbitMQ、MinIO 都要启动
- 版本不一致很容易出问题

Docker 的价值就是：

- 快速拉起依赖环境
- 保持环境一致
- 方便演示和交付

对于求职项目来说，Docker 是一个很加分的工程化能力。

## 2. 核心知识地图

### 2.1 镜像与容器

- 镜像：静态模板
- 容器：镜像运行后的实例

### 2.2 Dockerfile

作用：

- 定义你的应用镜像怎么构建

你项目里已经有：

- `Dockerfile`

### 2.3 Docker Compose

作用：

- 一次性编排多个依赖服务

你项目里已经有：

- `docker-compose.yml`

### 2.4 端口映射

例如：

- 容器 3306 映射到宿主机 13306
- 容器 8080 映射到宿主机 18081

### 2.5 环境变量

作用：

- 在不同环境下给应用注入不同配置

比如：

- 数据库地址
- Redis 地址
- MinIO 配置
- 大模型配置

## 3. MinIO 是什么

MinIO 可以理解成：

- 一个兼容 S3 的对象存储服务

对这个项目来说，它负责：

- 存储上传的文档文件

这比直接把文件丢本地磁盘更贴近真实企业系统。

## 6. 你要理解的工程化价值

### 6.1 为什么不只在 IDEA 里跑

IDEA 跑得起来，只能说明：

- 你本机临时环境 OK

Docker 跑得起来，才更像：

- 别人拿到项目也能复现
- 面试官看到你具备交付意识

### 6.2 为什么对象存储重要

知识平台本身就会接触：

- 文档上传
- 文档下载
- 预览
- 存储路径管理

所以 MinIO 这条线和你的业务是强相关的。

可以这样说：

“为了让项目更像真实交付系统，我把 MySQL、Redis、RabbitMQ、MinIO 都容器化了，并且给后端写了 Dockerfile，能直接构建和启动本地镜像。文档文件不再只存本地，而是通过 MinIO 接管存储，这样整个知识服务链路更接近企业生产环境。”
