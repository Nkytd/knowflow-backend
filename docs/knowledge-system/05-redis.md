# Redis 知识框架

## 1. Redis 在项目里解决什么问题

Redis 不是“有了更高级”，而是要明确它解决什么问题。

常见作用有：

- 缓存热点数据
- 提升读取速度
- 保存短期状态
- 作为分布式协作能力的一部分

对这个项目来说，Redis 可以进一步承担：

- 会话辅助缓存
- 检索缓存
- 统计指标缓存
- 临时任务状态缓存

## 2. 你要先建立的基础认知

Redis 本质上是：

- 基于内存的 Key-Value 数据库
- 非常快
- 支持多种数据结构

## 3. 核心知识地图

### 3.1 常见数据类型

- `String`
- `Hash`
- `List`
- `Set`
- `ZSet`

面试和项目里最常用的是：

- `String`
- `Hash`
- `ZSet`

### 3.2 为什么 Redis 快

- 内存访问快
- 数据结构高效
- 单线程模型减少上下文切换复杂度

### 3.3 缓存设计

你要理解：

- 什么数据适合缓存
- 缓存多长时间
- 缓存失效怎么办

### 3.4 常见缓存问题

- 缓存穿透
- 缓存击穿
- 缓存雪崩

这三个是 Redis 面试高频题。

## 4. 在这个项目里的可落地点

虽然当前项目重点不全在缓存，但 Redis 已经是架构组成部分，后续可以扩展：

### 4.1 问答结果短期缓存

适合缓存：

- 高频重复问题
- 某个知识库下的热门问答结果

### 4.2 看板指标缓存

适合缓存：

- 今日问答数
- 未命中率
- 工单转化率

### 4.3 临时上下文状态

比如：

- 某个用户最近活跃 session
- 某些短期任务标记

## 5. 项目里你要如何理解 Redis 的角色

不要把 Redis 理解为“必须什么都往里放”。

在企业项目里更合理的理解是：

- MySQL 负责持久化
- Redis 负责高频、短期、快速访问

## 6. Redis 与 MySQL 的分工

### MySQL 适合

- 核心业务数据
- 强一致数据
- 长期存储

### Redis 适合

- 热点读数据
- 临时状态
- 秒级或分钟级缓存

## 7. 常见面试题

- Redis 为什么快
- Redis 常见数据结构有哪些
- 缓存穿透、击穿、雪崩分别是什么
- Redis 和 MySQL 怎么配合
- 为什么 Redis 用单线程也很快
- 什么场景适合用 `zset`

## 8. 你如何结合项目回答

可以这样讲：

“在我的知识服务平台里，MySQL 负责持久化业务数据，Redis 主要承担高频访问场景下的加速能力。后续像问答结果缓存、运营看板指标缓存、短期任务状态缓存都适合放到 Redis 中。这样可以降低数据库压力，也让问答和看板响应更快。”
