# 项目面试表达与复习框架

## 1. 这份文档是干什么的

前面几篇文档解决的是“知识理解”。

这篇解决的是：

- 面试时怎么讲项目
- 复习时怎么抓重点
- 怎么从“做过项目”变成“能讲清项目”

## 2. 先把项目一句话讲清

一句话版本：

“这是一个面向企业内部知识服务场景的智能知识库与工单协同平台，支持文档接入、异步解析、检索问答、问答转工单、知识回流和运营分析。”

## 3. 再把项目拆成四层

### 第一层：业务目标

解决企业内部常见问题：

- 知识分散
- 问题重复咨询
- 未命中后缺少闭环
- 知识更新滞后

### 第二层：核心链路

- 知识库管理
- 文档解析与切片
- 问答检索
- 问答转工单
- 工单解决后回流知识草稿
- 看板分析与失败治理

### 第三层：技术实现

- Spring Boot
- MySQL + Flyway + MyBatis-Plus
- Spring Security + JWT
- Redis
- RabbitMQ
- MinIO
- Docker

### 第四层：亮点

- 多租户
- 完整鉴权
- 异步解析与消息流
- 死信治理和回放
- 知识服务闭环
- 可容器化部署

## 4. 面试时的标准讲法

### 4.1 项目背景

“这个项目面向企业内部知识服务场景，目标是把知识库、问答、工单和知识回流做成一个完整闭环。”

### 4.2 你的工作

“我主要参与并实现了知识库、文档、解析任务、鉴权链路、问答转工单、知识回流、运营看板、失败任务治理以及容器化验证这几条核心链路。”

### 4.3 技术方案

“后端采用 Spring Boot，数据层用 MySQL + Flyway + MyBatis-Plus，鉴权使用 Spring Security + JWT，多租户隔离贯穿核心业务。文档接入后通过异步任务完成解析和索引，RabbitMQ 用于消息驱动和失败治理，MinIO 负责文档存储，Docker 用于本地依赖和镜像验证。”

### 4.4 项目亮点

“和普通 CRUD 项目不同，这个系统不仅有后台管理能力，还有知识检索问答、未命中转工单、工单回流成知识草稿，以及死信回放和运营看板等更接近真实生产系统的能力。”

## 5. 面试官可能追问什么

### 业务追问

- 为什么要做问答转工单
- 知识回流为什么有价值
- 看板指标怎么定义

### 技术追问

- 为什么要异步解析
- 为什么用 RabbitMQ
- 为什么用 JWT
- 为什么要做多租户隔离
- 文档为什么存 MinIO
- 数据库结构怎么设计

### 工程追问

- 你如何验证项目可以运行
- 如何做 smoke test
- 容器化后带来了什么价值

## 6. 你当前阶段最该准备的面试重点

因为你是找实习，不需要一开始就把每个底层原理讲到专家级。

你更需要的是：

1. 把项目主线讲清楚
2. 把你负责的模块讲清楚
3. 把核心技术选择讲清楚
4. 把几个亮点讲清楚

## 7. 建议你重点背熟的五段话

### 第一段：项目背景

你要能 30 秒讲清这个平台干什么。

### 第二段：系统架构

你要能讲出：

- 管理台
- 后端服务
- 数据库
- Redis
- RabbitMQ
- MinIO

### 第三段：问答闭环

你要能讲出：

- 文档接入
- 解析切片
- 检索问答
- 未命中转工单
- 工单回流知识库

### 第四段：工程化能力

你要能讲出：

- Docker
- Flyway
- smoke test
- 死信回放

### 第五段：你学到了什么

你要能讲出：

- Java 后端分层开发
- 鉴权链路
- 异步任务
- 数据建模
- 容器化验证

## 8. 如何用这个项目反向学习

你后续可以这样复习：

1. 选一个技术点，比如 JWT
2. 去看对应知识文档
3. 再回到项目里找对应代码
4. 最后用一句面试表达把它讲出来

这会比单纯背八股更有效。

## 9. 建议你下一步继续补的三类材料

1. 面试题题库
2. 项目讲解稿
3. 一周一轮的复习计划

## 10. 过关标准

如果你能做到下面这些，项目就已经能支撑实习面试了：

- 3 分钟讲清项目全貌
- 1 分钟讲清某个模块
- 对核心技术有基本原理理解
- 能把项目亮点和普通 CRUD 区分开
